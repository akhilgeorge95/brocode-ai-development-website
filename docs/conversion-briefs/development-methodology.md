## 9. The Promise (H1 Direction)
A four-phase delivery methodology — **Discover (4 weeks) → Build (12–26 weeks in 2-week sprints) → Harden (4–6 weeks) → Run** — with named stage gates, RACI templates, fortnightly steerings, and a regulator-evidence pack produced at every gate, designed to survive a PMO audit and a TDRA / CBUAE / FSRA review.

## 10. The Unique Mechanism
- **Phase 1 — Discover (fixed 4 weeks):** week 1 stakeholder mapping and use-case scoring against the Brocode value-risk matrix; week 2 data-readiness audit and model-feasibility spike; week 3 reference architecture and SoW drafting; week 4 stage-gate review with the client steering committee. Deliverables: feasibility memo, target architecture, RACI, SoW with fixed scope and a unit-priced change-control framework.
- **Phase 2 — Build (12–26 weeks, 2-week sprints):** product-owner pair (Brocode delivery lead + client product owner), sprint demo every second Tuesday, RAID review every second Thursday, monthly steering. Sprint-zero is a 2-week stand-up of the dev environment, CI/CD, MLflow registry, eval harness, and the regulator-evidence template.
- **Phase 3 — Harden (4–6 weeks):** dedicated UAT, model-evaluation report, bias and fairness pack, security and penetration test, performance and load test, regulator dry-run, hand-over rehearsal.
- **Phase 4 — Run:** managed-service ramp with a fixed 90-day hyper-care, named SRE on-call rota, model-drift alerts wired to the client's ticketing system, quarterly model-review board.
- **Governance gates:** four named gates — G0 (kick-off), G1 (end of Discover), G2 (mid-Build production-readiness review), G3 (end of Harden, go-live approval). Each gate produces a regulator-evidence pack.
- **RACI patterns:** five named delivery patterns (greenfield model, model migration, RAG platform, agentic system, AI control-plane rollout) each shipped with a pre-built RACI the client PMO can edit on day one.
- **Tooling backbone:** Jira / Linear for sprint, Confluence / Notion for ADRs, GitHub Projects for engineering work, MLflow for model lineage, Lucid / Mermaid for architecture, Asana / MS Project for the PMO-side view.
- **The "PMO-Pack"**: a 22-document delivery-governance pack handed to the client PMO at G0 — SoW template, change-control template, RAID template, stage-gate checklist, regulator-evidence template, hand-over checklist, hyper-care exit criteria.
